This is a lot of information and it‚Äôs quite difficult to understand at first. Reading error messages takes practice. If you look carefully, you‚Äôll notice that the error message is divided into three parts, the error, a note, and a help.

The error tells us that we have tried to use a moved value, namely data on line 8, column 14 of the file src/main.rs. The note portion of the error message tells us why it was used up and shows us code from the standard library to explain why. Let‚Äôs skip this part for now.

By using &data rather than data, we‚Äôre instructing the loop to iterate over a reference to the Vec, rather than the Vec itself. The upshot is that we can now iterate multiple times.

The catch is that for this task, you may not use a for loop. Instead, you‚Äôll need to work with the iterator returned by data.iter() directly. Iterators in Rust work similarly to iterators in Java. To implement this as a Java method using iterators rather than a for loop we might use something like this.

int manualSum(ArrayList<Integer> data) {
    int sum = 0;
    Iterator<Integer> iter = data.iterator();
    while (iter.hasNext()) {
        sum += iter.next();
    }
    return sum;
}

Iterators in Rust don‚Äôt have separate has_next() and next() methods. Instead, there‚Äôs just a single next() method that returns an Option<T>. Every Option is either None or Some(blah) for some value blah. Here‚Äôs a simple example of using an Option.

let x = Some(10);
assert!(x.is_some());
assert!(!x.is_none());
assert_eq!(x.unwrap(), 10);

let y = None;
assert!(y.is_none());
assert!(!y.is_some());

We can use is_some() or is_none() to determine if an Option is a Some or a None. If we have a Some, then we can unwrap it by calling unwrap() which returns the data. If we try calling unwrap() on None, we‚Äôll get a runtime error.

Task 2.4

Strings in Rust are similar to vectors of characters except that, unfortunately, they‚Äôre slightly more difficult to use because text is inherently complicated than vectors.

The most obvious difficulty is you cannot access individual characters in the same way you‚Äôd access individual elements of a vector.

let x = vec![1, 2, 3];
assert_eq!(x[0], 1);

let y = "ABC";
assert_eq!(y[0], 'A'); // FAILS!

That last line fails because you cannot index into a string like this. The problem is that Rust stores all strings in the UTF-8 encoding. This is a variable-length encoding where different characters may take a different number of characters to encode. So if you want to get the 10 th character from a string, for example, you cannot easily figure out where in the encoded data for the string the 10 th character starts without starting at the beginning of the string.

This means that to work with characters, we‚Äôre going to need to use an iterator over the characters. Fortunately, the chars() function returns an iterator that returns each character.

let example = "Here is my string";
for ch in example.chars() {
    println!("{ch}");
}

Just as we‚Äôve made a distinction between Vec<i32> and &[i32] where the latter is a reference to the former, in Rust we have a String data type and a reference to a string &str. When we have a function that takes a reference to a string, we use an argument of type &str. Note that strings we create by enclosing text in quotation marks have type &str and not String. If we have a String and we want to pass it to a function that takes a &str, we get a reference to the string using & just as we did with vectors and reference to vectors.
